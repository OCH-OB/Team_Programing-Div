## Unit 別ストーリー一覧

### Unit 1: 事件化エンジン
- US-02（事件一覧）の事件データ提供
- US-03（事件詳細）の事件データ提供
- 直接担当ストーリーなし（他 Unit のデータ基盤）

### Unit 3: 匿名捜査員向けフロントエンド
- **US-01**: 匿名で参加開始する（オンボーディング・ハッシュ化）
- **US-02**: 事件一覧を確認する（一覧画面・新着順）
- **US-03**: 事件の詳細を確認する（詳細画面・集計表示）
- **US-04**: 捜査報告を投稿する（投稿フォーム・Exif 除去・3タップ）
- **US-05**: 役立っている通知を受け取る（通知一覧・未読バッジ）
- **US-07**: 他の捜査員の状況を確認する（選択式集計表示）

### Unit 4: 投稿受付・保存基盤
- **US-04**: 捜査報告を投稿する（API 側・DynamoDB 保存・S3 URL）

### Unit 5: 投稿整理AI + 地点統合
- **US-07**: 他の捜査員の状況を確認する（集計データ生成）
- **US-09**: 確認候補地点の詳細を確認する（AISummary・危険要因）
- **US-10**: 優先度分類を確認する（priority_class 計算）

### Unit 7: 役立っている通知エンジン + 進化予兆
- **US-05**: 役立っている通知を受け取る（通知生成・API）
- **US-06**: 特命捜査員への予兆を感じる（role_state 遷移・予兆通知文）

### Unit 9: 行政ダッシュボード
- **US-08**: 確認候補地点の一覧を確認する（一覧画面・認証）
- **US-09**: 確認候補地点の詳細を確認する（詳細画面）
- **US-10**: 優先度分類を確認する（優先度表示）
- **US-11**: レビュー状態を記録する（レビュー更新・ReviewLog）

### Unit 11: デモ用即時処理トリガー
- **US-12**: デモ用即時処理を実行する（DemoFunction・同期実行）
- **US-13**: 事前デモデータを投入する（シードスクリプト）
